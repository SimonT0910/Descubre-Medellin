Contexto:

Aplicación web llamada “Descubre Medellín”, una plataforma tipo blog/red social donde los usuarios pueden descubrir, compartir y comentar lugares interesantes del área metropolitana del Valle de Aburrá.

Los usuarios pueden:

-	Buscar lugares por nombre o descripción.
-	Filtrar lugares por categoría y rango de precio.
-	Ver la lista de lugares publicados.
-	Ver detalles completos de cada lugar.
-	Registrarse e iniciar sesión.
-	Publicar nuevos lugares.
-	Comentar y calificar lugares.
-	Guardar lugares como favoritos.

En esta etapa el proyecto solo se desarrollará la vista principal (Home) de la aplicación.

Requerimientos funcionales:

RF-01: Mostrar la pagina principal con un encabezado que contenga el nombre del sitio y un menú de navegación.
RF-02: Mostrar una barra de búsqueda que permita al usuario buscar lugares por nombre o descripción.
RF-03: Mostrar filtros para que el usuario pueda filtrar lugares por categoría y por rango de precio.
RF-04: Mostrar un listado de lugares utilizando tarjetas visuales.
RF-05: Mostrar un pie de página con información básica del sitio.

Cada tarjeta debe incluir:

-	Imagen del lugar.
-	Nombre del lugar.
-	Categoría.
-	Precio aproximado.
-	Botón para ver detalles.

Restricciones:

-	Usar HTML5 semántico (header, nav, main, section, article, footer).
-	No usar únicamente divs para toda la estructura.
-	Usar CSS3 para estilos y diseño.
-	No utilizar frameworks como Bootstrap o Tailwind.
-	No usar librerías externas.
-	El diseño debe ser simple, claro y organizado.

Stack tecnológico:

-	HTML5 semántico.
-	CSS3.
-	Layout usando Flexbox o Grid.
-	Compatible con navegador moderno.
-	Sin dependencias externas.

Criterio de éxito:

Al finalizar, el usuario debe poder:
1.	Abrir el archivo index.html en un navegador.
2.	Ver el header con navegación del sitio.
3.	Ver la barra de búsqueda y filtros.
4.	Visualizar una lista de lugares mostrados como tarjetas.
5.	Ver el footer del sitio correctamente estructurado.
