Registro de decisiones:
  1.	Restringir la carga de imágenes solo a archivos locales:

    	- Lo que hizo la IA: Permitía subir imágenes por URL o por archivo.

    	-	Mi cambio: Le pedí que quitara la URL y dejar solo cargar archivos desde el PC.

        - El porqué: Para simplificar la interfaz de usuario y asegurar que el proceso de "Publicar Lugar" sea más directo y realista para alguien que toma una foto con su dispositivo.
      
  2.	Corregir y completar el Filtro de Precios:
     
        -	Lo que hizo la IA: El filtro de precio inicial era básico, no filtraba realmente y solo mostraba opciones limitadas de precio.
      
        -	Mi cambio: Le pedí que el filtro funcionara por medio tres categorías de precio (bajo, medio, alto).
      
        -	El porqué: Porque un buscador no es útil si los filtros no funcionan. Era necesario que el sistema comparara el nivel de precio seleccionado con el del lugar publicado para que la experiencia de usuario fuera funcional (RF-03).
      
  3.	Conexión total del flujo de navegación:
     
        -	Lo que hizo la IA: Genero las páginas sueltas (scripts y HTML aislados) que no se conectaban entre sí.
      
        -	Mi cambio: Le pedi que todo estuviera enlazado (que el Login lleve al Home, que el Home lleve a Publicar y que al Publicar se vea el cambio en el Home).
      
        -	El porqué: Porque una aplicación web no es solo un diseño, es un flujo. Sin estas conexiones, el usuario se sentiría perdido en páginas "solitarias" y no podría completar el ciclo de descubrir y compartir lugares.
